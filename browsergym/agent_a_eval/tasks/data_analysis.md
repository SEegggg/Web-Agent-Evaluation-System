# 数据分析能力评审

## 可用数据集
- sales_2024.csv: 销售数据，包含 date/product/quantity/price/region 列
- customers.csv: 客户数据，包含 id/name/age/city/total_purchases 列
- iris.csv: 鸢尾花数据，包含 sepal_length/sepal_width/petal_length/petal_width/species 列

## 步骤

1. 上传本轮指定的数据集文件到 Agent A 平台。
2. 告诉 Agent A："请帮我分析这个数据集，重点查看各字段的分布、缺失值、异常值和字段之间的相关性，并把结论保存为分析报告"。
3. 等待 Agent A 完成分析。观察处理状态，确认分析过程已结束。
4. 确认页面上出现了分析报告内容（应包含数据概览、统计摘要、相关性分析等）。

## 评估标准

### 任务完成度 (权重 40%)
Agent A 是否成功生成了分析报告？若失败，完成到什么程度？
- 10分: 完整生成报告，包含数据概览、分布分析、缺失值/异常值检测、相关性分析
- 5分: 部分完成（如只有数据概览但缺少深度分析）
- 0分: 完全无响应或报告未生成

### 是否陷入死循环 (一票否决)
Agent A 是否陷入了重复输出、无响应、或无法停止的循环状态？
若检测到死循环，此项计 0 分且 overall_score 不超过 2 分。

### 分析质量 (权重 35%)
若报告成功生成，评估其：
- 数据分布描述的准确性（是否正确描述了均值、中位数、分位数等）
- 缺失值/异常值检测的完整性（是否发现了存在的问题）
- 关联分析的深度（是否给出了有意义的业务洞察）

### 代码与输出规范 (权重 25%)
- 代码是否清晰可读、有适当注释
- 输出格式是否规范、结构化
- 图表（如有）是否合适、标注完整
