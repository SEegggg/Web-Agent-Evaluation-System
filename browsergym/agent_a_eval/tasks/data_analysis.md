# 数据分析能力评审

## 可用数据集
- sales_2024.csv: 销售数据，包含 date/product/quantity/price/region 列
- customers.csv: 客户数据，包含 id/name/age/city/total_purchases 列
- iris.csv: 鸢尾花数据，包含 sepal_length/sepal_width/petal_length/petal_width/species 列

## 步骤

1. 上传本轮指定的数据集文件到 Agent A 平台。
2. 告诉 Agent A："请帮我分析这个数据集，重点查看各字段的分布、缺失值、异常值和字段之间的相关性，并把结论保存为分析报告"。
3. 等待 Agent A 完成分析。观察处理状态，确认分析过程已结束。
4. 确认页面上出现了分析报告内容（应包含数据概览、统计摘要、相关性分析等）。

## 评估标准

### 核心指标: 任务成功率（一票通过/否决）

**通过条件（必须全部满足）：**
- Agent A 成功生成了分析报告，报告包含数据概览（基本统计信息）
- 报告中包含分布分析（对各字段的分布描述）
- 报告中包含缺失值/异常值检测结果
- 报告中包含字段之间的相关性分析

**失败条件（满足任一即为失败）：**
- Agent A 完全无响应，未生成任何报告内容
- 报告仅包含数据概览，缺少分布分析、缺失值检测、相关性分析中的两项及以上
- Agent A 输出的分析结果存在明显逻辑错误（如把分类变量当数值变量做相关性分析）
- Agent A 使用了完全错误的分析方法导致结论无效

**归因说明：**
- 如果 Agent A 正确完成了分析流程但因数据集本身质量差导致结论有限 → PASS
- 如果驱动 Agent 提前终止但 Agent A 已生成完整报告 → PASS（失败归因于驱动 Agent）
- 如果 Agent A 遗漏了关键分析步骤（如跳过相关性分析）→ FAIL（归因于 Agent A）

### 任务专项指标

- **分析质量**: 数据分布描述是否准确（均值/中位数/分位数等统计量是否正确）、缺失值/异常值检测是否完整（是否发现了数据中实际存在的问题）、关联分析是否有深度（是否给出了有意义的业务洞察而非仅列出相关系数）
- **代码与输出规范**: 代码是否清晰可读有适当注释、输出格式是否规范结构化（如使用标题/表格清晰呈现）、图表（如有）是否合适且标注完整
