# 数据清洗能力评审

## 可用数据集
- dirty_sales.csv: 脏销售数据，包含缺失价格、重复行、异常数量的记录
- messy_customers.csv: 混乱客户数据，包含格式不一致的日期、拼写错误城市名、重复记录
- raw_iris.csv: 原始鸢尾花数据，包含部分缺失值、极端异常值

## 步骤

1. 上传本轮指定的脏数据集文件到 Agent A 平台。
2. 告诉 Agent A："请帮我清洗这个数据集：处理缺失值（数值列用中位数填充，分类列用众数填充）、删除完全重复的行、检测并标记异常值（使用 IQR 方法），保留所有字段，并将清洗后的数据保存并注册为数据集"。
3. 等待 Agent A 完成清洗处理。
4. 确认页面上出现了清洗后的数据预览或清洗完成的提示。

## 评估标准

### 核心指标: 任务成功率（一票通过/否决）

**通过条件（必须全部满足）：**
- Agent A 正确执行了缺失值处理（数值列用中位数填充，分类列用众数填充）
- Agent A 正确删除了完全重复的行
- Agent A 检测并标记了异常值（使用 IQR 方法或等效方法）
- 清洗后的数据保留了所有原始字段
- 清洗后的数据成功保存并注册为数据集

**失败条件（满足任一即为失败）：**
- Agent A 未执行任何清洗操作
- 缺失值处理方式错误（如数值列未用中位数、分类列未用众数）
- 未删除重复行或删除了非重复行
- 清洗后数据丢失了原始字段
- 清洗后的数据未保存或注册

**归因说明：**
- 如果数据集本身没有缺失值/重复行，Agent A 正确判断无需处理 → PASS
- 如果 Agent A 正确处理了所有清洗步骤但因平台问题数据注册失败 → PASS（归因于基础设施）
- 如果 Agent A 错误地删除了非重复行或用错误方法填充缺失值 → FAIL（归因于 Agent A）

### 任务专项指标

- **清洗正确性**: 缺失值处理是否正确（数值列中位数、分类列众数）、重复行是否被正确删除（去重前后行数合理）、异常值检测方法是否合理（IQR 参数是否正确）、清洗后的数据是否符合预期
- **代码质量与规范性**: 清洗代码是否清晰可复现、是否输出了清洗前后的数据对比（如行数变化、缺失值数量变化）、是否记录了清洗日志或处理步骤说明、清洗后的数据是否正确注册为数据集
