# 数据清洗能力评审

## 可用数据集
- dirty_sales.csv: 脏销售数据，包含缺失价格、重复行、异常数量的记录
- messy_customers.csv: 混乱客户数据，包含格式不一致的日期、拼写错误城市名、重复记录
- raw_iris.csv: 原始鸢尾花数据，包含部分缺失值、极端异常值

## 步骤

1. 上传本轮指定的脏数据集文件到 Agent A 平台。
2. 告诉 Agent A："请帮我清洗这个数据集：处理缺失值（数值列用中位数填充，分类列用众数填充）、删除完全重复的行、检测并标记异常值（使用 IQR 方法），保留所有字段，并将清洗后的数据保存并注册为数据集"。
3. 等待 Agent A 完成清洗处理。
4. 确认页面上出现了清洗后的数据预览或清洗完成的提示。

## 评估标准

### 任务完成度 (权重 40%)
Agent A 是否成功完成了数据清洗？若失败，完成到什么程度？
- 10分: 所有清洗步骤正确执行，清洗后的数据成功生成并注册
- 5分: 部分清洗完成（如处理了缺失值但未处理重复值）
- 0分: 未执行任何清洗操作

### 是否陷入死循环 (一票否决)
Agent A 是否陷入了重复输出、无响应、或无法停止的循环状态？
若检测到死循环，此项计 0 分且 overall_score 不超过 2 分。

### 清洗正确性 (权重 35%)
- 缺失值处理是否正确（数值列中位数、分类列众数）
- 重复行是否被正确删除
- 异常值检测方法是否合理
- 清洗后的数据行数/列数是否符合预期

### 代码质量与规范性 (权重 25%)
- 清洗代码是否清晰、可复现
- 是否输出了清洗前后的数据对比
- 是否记录了清洗日志或处理步骤说明
- 清洗后的数据是否正确注册为数据集
