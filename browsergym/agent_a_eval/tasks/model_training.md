# 模型训练能力评审

## 可用数据集
- iris.csv: 鸢尾花数据，目标列 species，三分类问题
- house_prices.csv: 房价数据，目标列 price，回归问题，包含 sqft/bedrooms/bathrooms/age/location 等特征
- customer_churn.csv: 客户流失数据，目标列 churn，二分类问题，包含 tenure/monthly_charges/contract_type/payment_method 等特征

## 步骤

1. 上传本轮指定的数据集文件到 Agent A 平台。
2. 告诉 Agent A："请基于这个数据集训练一个分类或回归模型（根据目标列类型选择），优先使用简单可解释的基线模型（如决策树或线性回归），输出完整的评估指标（分类：准确率/精确率/召回率/F1；回归：RMSE/MAE/R²），并将训练好的模型保存并注册为模型资产"。
3. 等待 Agent A 完成模型训练（包括数据预处理、特征工程、模型训练、评估）。
4. 确认页面上出现了模型评估结果。

## 评估标准

### 核心指标: 任务成功率（一票通过/否决）

**通过条件（必须全部满足）：**
- Agent A 正确识别了目标列类型并选择了匹配的模型类型（分类任务选分类器，回归任务选回归器）
- Agent A 完成了模型训练全流程（数据预处理 → 特征工程 → 训练 → 评估）
- Agent A 输出了完整的评估指标（分类：准确率/精确率/召回率/F1；回归：RMSE/MAE/R²）
- 训练好的模型成功保存并注册为模型资产
- 优先使用了简单可解释的基线模型（如决策树或线性回归）

**失败条件（满足任一即为失败）：**
- 模型训练未启动或训练失败
- 模型类型与任务类型不匹配（如对回归任务使用了分类模型）
- 未输出要求的评估指标（缺失超过一项核心指标）
- 模型未保存或注册
- 使用了明显不合适的复杂模型而拒绝了要求的简单基线模型

**归因说明：**
- 如果 Agent A 正确完成了训练全流程但因数据集质量差导致模型效果不好 → PASS
- 如果 Agent A 正确完成训练但人类指定的评估指标不适用于该任务类型 → PASS
- 如果 Agent A 遗漏了数据预处理步骤（如未做标准化/编码）→ FAIL（归因于 Agent A）
- 如果 Agent A 未按要求使用基线模型而是用了复杂黑盒模型 → FAIL（归因于 Agent A）

### 任务专项指标

- **模型选择合理性**: 是否按要求选择了简单可解释的基线模型、模型类型与任务类型是否匹配、是否有基本的模型选择理由说明
- **评估完整性**: 是否输出了所有要求的评估指标、指标计算是否正确、是否有训练集/测试集的划分
- **代码与输出规范**: 训练代码是否包含必要的预处理步骤（编码、标准化等）、输出格式是否规范且包含模型基本信息、模型文件是否正确保存和注册
