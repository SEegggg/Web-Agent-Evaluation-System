# 数据注册

## 任务描述（可选）

本任务评估 Agent A 将工作空间（workspace）中的 CSV 文件正确注册为数据资产的能力。数据注册是数据科学工作流中的基础环节，要求 Agent A 能够识别文件、解析结构、提取元数据（表名、列名、数据类型），并将该文件作为正式资产录入数据目录。通过此任务，检验 Agent A 在数据资产化管理方面的自动化水平与规范遵循度。

## 可用数据集

- sales_2024.csv: 销售数据，包含 date（日期）、product（产品名）、quantity（销售量）、price（单价）、region（区域）共5列，存在少量缺失值。
- iris.csv: 鸢尾花数据，包含 sepal_length、sepal_width、petal_length、petal_width、species 共5列，无缺失值，species 为分类列。

## 步骤

1. 将本轮指定的数据集文件（从 available datasets 中选择一个）上传至 Agent A 的 workspace 中。
2. 向 Agent A 下达指令：“请将 workspace 中的 ```{filename}``` 注册为数据资产，要求自动识别列名和数据类型，并设置一个合适的表名。”
3. 等待 Agent A 完成注册流程（观察任务状态或页面提示）。注册完成后，检查数据资产列表或目录中是否出现对应的资产记录。
4. 如果 Agent A 提供了注册结果摘要（如列名列表、数据类型推断结果），请截图或记录以备评估。

## 评估标准

### 核心指标: 任务成功率（一票通过/否决）

**通过条件（必须全部满足）：**
- Agent A 成功执行了注册操作，工作空间中创建的对应数据资产（data asset）记录明确显示文件已被注册。
- 数据资产记录中包含正确的表名（表名应基于文件名或用户给定名称合理设置，且不与其他资产冲突）。
- 注册结果中列出了文件的所有列名，且数量与原始 CSV 一致（无遗漏或多余列）。

**失败条件（满足任一即为失败）：**
- Agent A 完全未触发注册动作（例如仅回复了操作说明但未实际执行）。
- 注册过程报错且 Agent A 未采取任何补救尝试，导致最终无数据资产生成。
- 生成的数据资产记录中列数量与原始 CSV 不一致（例如 missing columns 或额外加入非 CSV 列）。
- Agent A 将文件内容直接展示但未执行注册（混淆了“查看”与“注册”操作）。

**归因说明：**
- 若 Agent A 正确完成了注册，但由于上传的文件本身损坏或格式不标准导致注册后部分元数据异常（如日期列被误判为字符串） → PASS（归因于文件质量，而非 Agent A）。
- 若 Agent A 因网络超时或平台内部错误未能完成注册，且无自主重试 → FAIL（归因于 Agent A 未处理异常）。
- 若 Agent A 注册了数据资产但遗漏了关键列（如忽略包含缺失值的列） → FAIL（归因于 Agent A 解析错误）。

### 任务专项指标

- **注册完整性**: Agent A 是否完整提取了所有列的信息（列名、推断的数据类型），以及是否对分类变量（如 species）给出了正确的类型识别（例如识别为“categorical”或“string”而非“number”）。评分标准：
  - 优秀：所有列的类型推断基本准确，分类列被正确识别，无列遗漏。
  - 合格：大部分列类型正确，但有1-2列类型偏差（如日期列被识别为字符串）。
  - 不合格：超过2列类型错误，或遗漏了至少1列。

- **元数据丰富度**: 注册后的数据资产是否包含额外的描述性元数据（如表注释、字段注释、行数统计、缺失值统计等）。Agent A 应主动提供此类信息以提升资产可用性。评分标准：
  - 优秀：自动生成了完整的数据摘要（行数、列数、缺失值统计、各列数据类型）并作为元数据关联。
  - 合格：提供了基本的元数据（如表名、列数、部分列类型），但缺乏缺失值统计或描述性注释。
  - 不合格：仅注册了表名和列名，无任何额外元数据。

- **错误处理与用户反馈**: 当遇到潜在问题（如列名包含特殊字符、数据类型不明确、缺失值比例高）时，Agent A 是否给出明确的警告或建议，并在注册前征得用户确认（可选）。若用户未指定表名，Agent A 是否合理生成默认名。评分标准：
  - 优秀：主动检查数据质量问题并给出透明提示，用户可根据提示调整。
  - 合格：注册成功但未提示任何数据质量信息。
  - 不合格：注册过程中遇到问题但未报错、未提示，直接以错误配置完成注册。
